# 问题解决文档：阅读题英语单词显示编码问题修复

## 问题概述

**问题描述**：个别学生在做阅读题的时候看到的是编码，不是英语单词

**影响范围**：阅读题模块，影响学生正常答题体验

**问题类型**：字符编码处理问题

**发现时间**：2025年9月28日

**解决时间**：2025年9月28日

## 问题分析

### 技术根因

1. **DOMParser 字符编码缺失**
   - 使用 `DOMParser.parseFromString()` 时未指定字符编码
   - 导致非ASCII字符（如英语单词中的特殊字符）解析错误

2. **不安全的HTML文本提取**
   - 使用正则表达式 `replace(/<[^>]+>/g, "")` 处理HTML
   - 可能破坏字符编码结构

3. **HTML实体处理不完整**
   - 仅处理 `&nbsp;` 实体
   - 忽略其他常见HTML实体如 `&amp;`, `&lt;`, `&gt;` 等

### 影响的文件

- `src/components/basic/questions/index.tsx` - 主要问题文件
- `src/components/basic/readQuestions/index.tsx` - 相关组件
- `src/components/basic/listenQuestions/index.tsx` - 相关组件

## 解决方案

### 1. 修复 `stripHtmlTags` 函数

**问题代码**：
```javascript
const stripHtmlTags = (html: string): string => {
  return html.replace(/<[^>]+>/g, "").replace(/&nbsp;/g, " ");
};
```

**修复后代码**：
```javascript
const stripHtmlTags = (html: string): string => {
  // 创建一个临时的 DOM 元素来安全地解析 HTML
  const tempDiv = document.createElement('div');
  tempDiv.innerHTML = html;
  
  // 获取纯文本内容，这样可以正确处理字符编码
  const textContent = tempDiv.textContent || tempDiv.innerText || '';
  
  // 清理常见的 HTML 实体
  return textContent
    .replace(/&nbsp;/g, ' ')
    .replace(/&amp;/g, '&')
    .replace(/&lt;/g, '<')
    .replace(/&gt;/g, '>')
    .replace(/&quot;/g, '"')
    .replace(/&#39;/g, "'");
};
```

**改进要点**：
- 使用 DOM API 而非正则表达式
- 利用 `textContent` 安全提取文本
- 完整处理HTML实体字符

### 2. 修复 `replaceFontSize` 函数

**关键改进**：

```javascript
const replaceFontSize = (html: string, fontSize: number): string => {

  const parser = new DOMParser();
  const doc = parser.parseFromString(htmlWithMeta, "text/html");

```

**改进要点**：
- 添加 UTF-8 字符编码声明
- 确保 DOMParser 正确解析字符
